# 기술 스택

이 프로젝트는 다음 기술 스택을 사용합니다.
- Vite
- TypeScript
- React
- shadcn-ui
- Tailwind CSS


# 개발 프로세스
1. 테마 설정: 사용자 요구사항을 참고하여 src/index.css와 tailwind.config.ts의 테마 스타일을 조정합니다.
2. 페이지 설계: 사용자 요구사항에 따라 구현이 필요한 페이지를 구분합니다.
3. 페이지 생성: 각 페이지별로 구현해야 할 기능을 정리하고, pages 디렉토리 하위에 해당 폴더와 입구(Entry) 파일인 Index.tsx를 생성합니다.
4. 라우팅 설정: App.tsx에서 라우팅 설정을 생성하고, 방금 만든 각 페이지의 Index.tsx 파일들을 연결합니다.
5. 단순 페이지 구현: 요구사항이 간단한 경우, Index.tsx 안에서 해당 페이지의 모든 작업을 직접 완료할 수 있습니다.
6. 복잡 페이지 구조화: 요구사항이 복잡할 경우, 페이지를 여러 컴포넌트로 분리하여 구현하며 디렉토리 구조는 다음과 같이 구성합니다.
    - Index.tsx 입구 파일
    - /components/ 컴포넌트모음
    - /hooks/ 커스텀 훅
    - /stores/ 복잡한 상호작용이나 상태 공유가 필요한 경우 Zustand를 사용하여 통신
7. 검사 및 수정: 요구사항 구현이 끝나면 pnpm i로 의존성을 설치하고, npm run lint 및 npx tsc --noEmit -p tsconfig.app.json --strict 명령어를 실행하여 문제를 점검하고 수정합니다.

# 백엔드 인터페이스 연동
- API 추가: 새로운 인터페이스를 추가하거나 Supabase를 조작해야 할 경우, 우선 src/api에 대응하는 API 파일을 신규 생성하고 데이터 타입을 export 합니다. (참조: src/demo.ts) Supabase를 사용하는 경우 관련 구현까지 완료해야 합니다.
- 타입 준수: 프론트엔드와 Supabase를 연동할 때는 정의된 데이터 타입을 엄격히 준수해야 합니다. 이미 정해진 데이터 타입을 수정하는 것은 가급적 피해야 하며, 만약 수정이 발생할 경우 해당 타입을 인용하는 모든 파일을 점검해야 합니다.
